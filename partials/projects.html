Sleep Apnea Detection

Role: Software Developer Intern
This project is an initiative of MIT Media Labs and focuses on building a device in form of wearable mask to collect sleep data. This data can then be analysed using Machine Learning algorithms to find Apneic and Hypo-apneic events. Based on further investigation, the subject can be diagnosed for Obtrusive Sleep Apnea.

Sign Language Recognition using Convolutional Neural Networks

Team: Anirudh Nihalani, Brij Mohan, Mallikarjun
This work focuses on exploiting and comparing various combination of techniques to enhance the accuracy of Sign Language Recognition. We conducted several experiments using varying architectures of CNN, pre-processing of data and inclusion of "good" features. The goal is to study the effects of different parameters on the accuracy and report the combinations which gave best results.

Comparison of Ranking Algorithms (TF-IDF, Google Pagerank and Lucene)

Team: Brij Mohan, Neel Jinwala, Prachish Gora
Problem Statement:
A thorough analysis of different ranking algorithms and applying them. Similar to the search engine that has been implemented but involves extended and optimized methodologies and comparing the output with standards like Lucene
Motivation:
There are several ranking algorithms that are used for various search domains. Each algorithm produces best results when subjected to a certain type of data. We have taken up two of those popular algorithms in order to find their best use-cases and loopholes in order to compare them with the standard lucene engine. This may also help us find their best possible scenarios and come up with something unique and ubiquitous.
Approach:
There are several modules involved in the implementation of the project.
  1. Creation of Inverted index based on the new approach, which involves storing additional document
    2. information such as, outlinks, page rank of the document, additional factors to calculate local weightage, etc.
  2. Intermediate outputs involving recursive calculation of page rank for every document.
  3. Generation of secondary index along with document mapping index.
  4. Compression of primary index
  5. Efficient query searching along with caching
We will be covering two ranking algorithms namely, tf-idf (term frequency-inverse document frequency) and Google's Page Rank algorithm devised by Sergey Brin and Larry Page.

Wikipedia 43GB Corpus
